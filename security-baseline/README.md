# 🔐 Security Baseline & Endpoint Hardening Lab

## 📌 Objective

The purpose of this lab was to implement security best practices in a Microsoft 365 and Windows environment to protect devices, users, and data.

This includes enforcing security controls such as MFA, Conditional Access, BitLocker, and endpoint security configurations.

---

## 🧩 Scenario

An organisation needs to improve its security posture and reduce risk by implementing modern security controls based on Zero Trust principles.

As the IT Administrator, I configured security settings to protect user identities, secure endpoints, and enforce compliance across devices.

---

## 🛠️ Technologies Used

- Microsoft 365 Admin Center
- Microsoft Intune
- Microsoft Entra ID (Azure AD)
- Windows 11
- BitLocker

---

## ⚙️ What I Configured

### 1. Multi-Factor Authentication (MFA)
- Enabled MFA for user accounts
- Strengthened identity security
- Reduced risk of unauthorised access

---

### 2. Conditional Access Policies
- Created access control rules based on:
  - User identity
  - Device compliance
- Enforced secure login conditions

---

### 3. Windows Security Baselines
- Applied recommended Microsoft security configurations
- Strengthened endpoint protection settings

---

### 4. BitLocker Configuration
- Enabled disk encryption on Windows devices
- Protected data at rest

---

### 5. Secure Score Improvements
- Reviewed security recommendations
- Implemented improvements to increase security posture

---

## ✅ Results

- Stronger identity protection using MFA  
- Controlled access through Conditional Access policies  
- Devices secured using Microsoft security baselines  
- Data protected with BitLocker encryption  
- Improved overall Microsoft Secure Score
